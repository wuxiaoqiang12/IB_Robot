# AtomGit PR 代码审查 Skill

## 概述

代码审查 skill 用于自动审查 AtomGit 仓库的 Pull Request。本 skill 基于官方 code-review 插件的 9 步流程设计，采用多代理并行审查、独立验证和置信度评分机制，确保只报告高质量的问题。

### 审查维度

1. **Bug 扫描** - 扫描明显的语法错误、类型错误、API 误用
2. **代码问题分析** - 查找安全问题、逻辑错误、性能问题、错误处理缺陷
3. **语义深度分析** - 理解代码意图和上下文，发现深层逻辑问题

### 核心特性

- **置信度评分** - 只发布置信度 >= 80 的问题
- **自动去重** - 相同问题只报告一次
- **AI 签名** - 每条评论包含 "generated by ai@claude"
- **官方参考资料** - 自动添加相关文档链接

## When to Use

Invoke this skill when:
- ✅ 审查 AtomGit 仓库的 Pull Request
- ✅ 自动发现代码中的 bug 和逻辑错误
- ✅ 生成高质量的代码审查意见
- ✅ 需要提交代码审查评论到 AtomGit PRs

Do NOT invoke for:
- ❌ 架构审查 (使用 `atomgit-architecture-review` skill)
- ❌ 语法或 lint 检查 (使用 linters)
- ❌ 安全审计 (使用 dedicated security tools)

## 使用方式

### 前置条件

1. Python 3.10+ 已安装
2. `requests` 库已安装 (`pip install requests`)
3. AtomGit Personal Access Token 已配置

### 配置

在项目根目录 (`/home/xqw/Research/IB_Robot/config.json`) 创建配置文件：

```json
{
  "atomgit": {
    "token": "your_personal_access_token",
    "owner": "openEuler",
    "repo": "IB_Robot",
    "baseUrl": "https://api.atomgit.com"
  },
  "codeReview": {
    "confidenceThreshold": 80
  }
}
```

### 获取 AtomGit Personal Access Token

1. 登录 [https://atomgit.com](https://atomgit.com)
2. 点击右上角头像 → 个人设置
3. 找到「访问令牌」选项
4. 创建新令牌，勾选 `repo` 和 `pull_request` 权限
5. 将 Token 填入 `config.json`

### 运行审查

**IMPORTANT**: 所有命令必须从项目根目录运行，并使用环境设置：

```bash
# 切换到项目根目录
cd /home/xqw/Research/IB_Robot

# 提取 PR 信息（供手动审查）
source .shrc_local && python3 .agents/skills/atomgit-code-review/scripts/atomgit_reviewer.py --pr 123

# 从 JSON 文件提交评论
source .shrc_local && python3 .agents/skills/atomgit-code-review/scripts/atomgit_reviewer.py --pr 123 --issues-from-json issues.json

# Dry run 模式（仅显示计划）
source .shrc_local && python3 .agents/skills/atomgit-code-review/scripts/atomgit_reviewer.py --pr 123 --issues-from-json issues.json --dry-run

# 自定义置信度阈值
source .shrc_local && python3 .agents/skills/atomgit-code-review/scripts/atomgit_reviewer.py --pr 123 --issues-from-json issues.json --threshold 70
```

## JSON 文件格式

创建 `issues.json` 文件：

```json
[
  {
    "file": "examples/lekiwi/evaluate_dataset.py",
    "line": 15,
    "type": "bug",
    "severity": "error",
    "confidence": 90,
    "title": "Shebang 格式错误",
    "description": "Shebang 缺少 ! 符号，导致脚本无法直接执行",
    "contextCode": "# !/usr/bin/env python\n# 应改为:\n#!/usr/bin/env python",
    "fix": {
      "code": "#!/usr/bin/env python",
      "explanation": "修复 shebang 格式，确保 # 和 ! 之间没有空格"
    }
  }
]
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file | string | 是 | 文件路径（相对于仓库根目录） |
| line | number | 是 | 问题行号（在新文件中的行号） |
| type | string | 是 | 问题类型：bug, security, logic_error, api_misuse |
| severity | string | 是 | 严重程度：error, warning |
| confidence | number | 是 | 置信度 0-100，只报告 >= 80 |
| title | string | 是 | 问题标题 |
| description | string | 是 | 详细描述 |
| contextCode | string | 是 | 上下文代码 |
| fix | object | 是 | 修复方案（code + explanation） |

## 审查规则

### 高信度问题标准

只报告满足以下条件的问题：

1. **编译/解析错误** - 语法错误、类型错误、缺少导入
2. **明确的逻辑错误** - 无论如何输入都会产生错误结果
3. **安全问题** - SQL 注入、XSS、硬编码敏感信息
4. **明显的 API 误用** - 不符合标准用法
5. **严重的错误处理问题** - 空 catch 块、吞掉异常

### 不报告的情况

- 代码风格问题
- linter 可捕获的问题
- 预先存在的问题
- 可能不是问题的地方（不确定时不报告）

## Troubleshooting

### Issue: ModuleNotFoundError when running review script

**Root Cause**: Environment not set up properly

**Solution**:
```bash
cd /home/xqw/Research/IB_Robot
source .shrc_local && python3 .agents/skills/atomgit-code-review/scripts/atomgit_reviewer.py --pr 123
```

### Issue: Authentication failed (401 error)

**Root Cause**: Invalid or missing AtomGit token

**Solution**:
1. 验证 `config.json` 在项目根目录存在
2. 检查 token 有 `repo` 和 `pull_request` 权限
3. 确保 `baseUrl` 正确 (`https://api.atomgit.com`)

### Issue: PR not found (404 error)

**Root Cause**: Wrong PR number or repository configuration

**Solution**:
1. 验证 PR 号在 AtomGit 上存在
2. 检查 `config.json` 中 `owner` 和 `repo` 匹配实际仓库
3. 确保 token 有仓库访问权限

### Issue: No issues submitted but expected

**Root Cause**: Confidence threshold filtering or no valid issues

**Solution**:
1. 使用 `--dry-run` 查看提交计划
2. 检查 JSON 文件中 confidence 值是否 >= 阈值
3. 使用 `--threshold 70` 降低阈值
