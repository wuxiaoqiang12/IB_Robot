# IB_Robot Architecture Review Skill

This skill provides automated architecture review for IB_Robot PRs on AtomGit platform.

**Prerequisites**: This skill works with `ibrobot-architecture` skill for architectural knowledge.

## Review Dimensions

1. **Spec-Driven Configuration**:
   - Check if parameters are hardcoded.
   - Check if configuration is managed by `robot_config`, not scattered in scripts.
2. **Layered Decoupling**:
   - Check if inference logic brings in development dependencies.
   - Check if code is placed in the wrong package (e.g., ONNX conversion should be in `model_tools` or `dataset_tools`).
3. **Data Flow Standards**:
   - Check if ROS 2 topic and LeRobot tensor conversion follows `tensormsg` hub.
4. **ROS 2 Native Integration**:
   - Check if Python scripts are properly wrapped as `rclpy.Node`.
   - Check if standard parameter declaration (`declare_parameter`) is used.

## When to Use

Invoke this skill when:
- ✅ Reviewing large feature PRs for architecture debt
- ✅ Verifying new modules comply with project layering principles
- ✅ Checking for "hardcoding" and "inter-module coupling" issues
- ✅ Validating code changes against IB_Robot architectural pillars
- ✅ Need to submit architecture review comments to AtomGit PRs

Do NOT invoke for:
- ❌ General code review (use standard code review tools)
- ❌ Syntax or lint checks (use linters)
- ❌ Security audits (use dedicated security tools)

## 9-Step Architecture Audit Process

```
Step 1: Scan Package Structure  → Which layer does the changed file belong to (application, planning, inference, control)?
Step 2: Dependency Audit         → Are there unnecessary cross-layer dependencies?
Step 3: Configuration Review     → Find hardcoded thresholds, paths, and device indices.
Step 4: ROS Alignment Check      → Is it a standard Node or an isolated script?
Step 5: Architecture Violation   → Does it break robot_config or tensormsg contracts?
Step 6-9: Summary & Publish      → Propose architecture-level modifications.
```

## Usage

### Prerequisites

Before using this skill, ensure:
1. Python 3.10+ is installed
2. Project dependencies are available (run `source .shrc_local` first)
3. AtomGit Personal Access Token is configured

### Configuration

Create `config.json` in project root (`/home/xqw/Research/IB_Robot/config.json`):

```json
{
  "atomgit": {
    "token": "your_personal_access_token",
    "owner": "openEuler",
    "repo": "IB_Robot",
    "baseUrl": "https://api.atomgit.com"
  }
}
```

### Run Review

**IMPORTANT**: All commands must be run from project root with environment setup:

```bash
# Change to project root first
cd /home/xqw/Research/IB_Robot

# Review PR and submit comments
source .shrc_local && python3 .agents/skills/atomgit-architecture-review/scripts/atomgit_reviewer.py --pr 123

# Dry run (generate report only, don't submit)
source .shrc_local && python3 .agents/skills/atomgit-architecture-review/scripts/atomgit_reviewer.py --pr 123 --dry-run
```

## Troubleshooting

### Issue: ModuleNotFoundError when running review script

**Root Cause**: Environment not set up properly

**Solution**:
```bash
cd /home/xqw/Research/IB_Robot
source .shrc_local && python3 skills/atomgit-architecture-review/scripts/atomgit_reviewer.py --pr 123
```

### Issue: Authentication failed (401 error)

**Root Cause**: Invalid or missing AtomGit token

**Solution**:
1. Verify `config.json` exists in project root
2. Check token has `repo` and `pull_request` scopes
3. Ensure `baseUrl` is correct (`https://api.atomgit.com`)

### Issue: PR not found (404 error)

**Root Cause**: Wrong PR number or repository configuration

**Solution**:
1. Verify PR number exists on AtomGit
2. Check `owner` and `repo` in `config.json` match actual repository
3. Ensure token has access to the repository

### Issue: No architecture violations found but expected

**Root Cause**: Review dimensions may not cover the specific issue type

**Solution**:
1. Run with `--dry-run` to see raw analysis
2. Check if issue falls under the 4 review dimensions
3. Consider extending review criteria in the script
