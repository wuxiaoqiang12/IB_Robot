# AtomGit PR Review Comment Repair

自动修复 AtomGit PR 检视意见，使用 LLM 生成修复方案。

## When to Use

Invoke this skill when:
- ✅ 用户提供 AtomGit PR 链接且有待解决的检视意见
- ✅ 用户想自动修复代码审查反馈
- ✅ 用户提到 "修复检视意见" 或类似表述
- ✅ 用户需要系统性地回复多条检视意见
- ✅ 用户想自动化 修复-回复-提交 工作流

Do NOT invoke for:
- ❌ 创建检视意见 (使用 `atomgit-code-review` skill)
- ❌ 架构审查 (使用 `atomgit-architecture-review` skill)
- ❌ CI/CD 修复 (使用专门的 CI 修复工具)

## Workflow

修复流程遵循以下步骤：

1. **解析 PR 链接** - 从 URL 提取 owner, repo, PR 号
2. **获取检视状态** - 获取已解决/总数评论数
3. **计算未解决** - `unresolved = total - resolved`
4. **检查完成** - 如全部解决，退出并显示成功消息
5. **获取未解决评论** - 获取每条检视意见的文件、行号、内容
6. **生成修复** (对每条评论):
   - 使用 Anthropic Claude 分析代码和检视意见
   - 生成修复方案 或 检测检视逻辑缺陷
   - 准备结构化回复及修复详情
7. **回复评论** - 通过 AtomGit API 发布回复
8. **应用代码修复** - 根据修复建议编辑文件
9. **提交** - 使用 `git commit --amend` 更新提交
10. **输出汇总** - 显示所有修复和评论链接的表格

```

[Brief description of the fix]

**上下文代码**:
```
# Before
[original code]

```
**关于此检视意见的说明**:

如有任何疑问，请随时提出。感谢您的审阅！

---
🤖 Generated by atomgit-code-review-repair
```

## Configuration

Requires `config.json` in project root:

```json
{
  "atomgit": {
    "token": "your_atomgit_token",
    "owner": "openEuler",
    "repo": "IB_Robot",
    "baseUrl": "https://api.atomgit.com"
  },
  "anthropic": {
    "apiKey": "sk-ant-..."
  }
}
```

**Note**: Anthropic API key is required for LLM-based fix generation.

## Usage

### Command Line

**IMPORTANT**: All commands must be run from project root with environment setup:

```bash
# 切换到项目根目录
cd /home/xqw/Research/IB_Robot

# 运行修复脚本
source .shrc_local && python3 .agents/skills/atomgit-code-review-repair/scripts/repair_pr.py

# 或直接提供 PR URL
source .shrc_local && python3 .agents/skills/atomgit-code-review-repair/scripts/repair_pr.py https://atomgit.com/openEuler/IB_Robot/pull/50
```

### Interactive Mode

The script will prompt for:
1. PR URL (if not provided via command line)
2. Solution selection (choose from multiple LLM-generated solutions)
3. Confirmation before applying each fix

## LLM Integration

The skill uses Anthropic Claude to:

1. **Analyze** the review comment and surrounding code context
2. **Determine** if the review is valid or if the logic is flawed
3. **Generate** precise code fixes with before/after comparison
4. **Explain** the reasoning for fixes or disputes

### Response Types

The LLM can generate:
- `HAS_FIX`: Code modification needed
- `NEEDS_REVERT_FILE`: Entire file should be reverted
- `NEEDS_DELETE_LINES`: Specific lines should be deleted
- `NEEDS_REPLY_ONLY`: Only clarification reply needed

## Troubleshooting

### Issue: ModuleNotFoundError when running script

**Root Cause**: Environment not set up properly

**Solution**:
```bash
cd /home/xqw/Research/IB_Robot
source .shrc_local && python3 .agents/skills/atomgit-code-review-repair/scripts/repair_pr.py
```

### Issue: Authentication failed (401 error)

**Root Cause**: Invalid or missing AtomGit token

**Solution**:
1. Verify `config.json` exists in project root
2. Check token has `repo` and `pull_request` scopes
3. Ensure `baseUrl` is correct (`https://api.atomgit.com`)

### Issue: LLM API Error

**Root Cause**: Missing or invalid Anthropic API key

**Solution**: Add `anthropic.apiKey` to config.json:
```json
{
  "anthropic": {
    "apiKey": "sk-ant-..."
  }
}
```

### Issue: Git Amend Error

**Root Cause**: Not in git repository or no changes to amend

**Solution**: Ensure:
- You're in a git repository
- You have uncommitted changes to amend
- You're on the correct branch
